# GoDesk 设计方案

> 状态：设计定稿，已完成用户审阅  
> 设计版本：1.0  
> 更新日期：2026-07-22  
> 适用范围：两台自用 Windows 电脑之间的远程控制与文件传输
> 扩展规格：[Codex MCP 与持久远程终端设计](./2026-07-22-godesk-codex-mcp-terminal-design.md)

## 1. 项目目标

GoDesk 是一款安全优先、无需自建公网服务器的 Windows 远程控制软件。它面向两台个人电脑，支持局域网直连和跨互联网连接。

第一版目标：

- 远程桌面画面；
- 键盘和鼠标控制；
- 文件传输；
- 多显示器切换；
- 远程系统声音；
- 开机自启；
- 配对后的无人值守访问；
- 内置持久远程终端和 A 端 Codex MCP；
- 局域网 `1080p 60fps`；
- 互联网自适应至 `1080p 30fps`；
- 不要求用户租用或维护 Ubuntu 服务器。

## 2. 范围与约束

### 2.1 第一版支持

- Windows 11 22H2 及以上；
- x64；
- 两台电脑一对一使用；
- 同一局域网直接连接；
- 通过 Tailscale 虚拟网络跨互联网连接；
- 一次传输一个远端显示器，允许随时切换；
- PowerShell、PowerShell 7、CMD、ConPTY 和既有 Conda 环境；
- A 端本地 STDIO MCP Bridge；
- 有活动物理显示输出的电脑。

### 2.2 第一版不支持

- Windows 10；
- macOS、Linux、Android 和 iOS；
- 浏览器控制端；
- 多人同时控制；
- 云端设备列表；
- 摄像头和语音对讲；
- TCP 隧道、SSH 服务和通用端口转发；
- 自动静默升级；
- 同时展示所有远端显示器；
- 修改远端显示器排列；
- 自动安装虚拟显示器驱动；
- 没有活动显示输出的纯无头主机；
- 隐藏远控状态或静默监控。

## 3. 开源项目调研结论

调研过的主要项目包括 RustDesk、CrossDesk、MeshCentral、Sunshine、P2P Remote Desktop、Remotely、IronRDP 和 FreeRDP。

结论：

- RustDesk 与目标功能最接近，适合作为行为、部署和协议设计参考；
- CrossDesk 的 MiniRTC、音视频和 Windows 服务设计值得研究，但项目仍具有实验性质；
- MeshCentral 更偏远程设备运维，不以低延迟声音和画面为核心；
- Sunshine 更偏游戏串流，缺少完整的文件传输和自动中继体系；
- 简单 P2P 项目普遍缺少可靠的 NAT 穿透、认证和无人值守安全模型；
- GoDesk 不复制 RustDesk 的 AGPL 代码，以避免许可证传染和大型上游代码维护负担；
- 可以参考协议思想和公开架构，但独立实现 GoDesk 代码。

## 4. 网络方案

### 4.1 总体策略

GoDesk 不部署自己的公网协调和中继服务器，使用以下连接顺序：

```text
同一局域网地址直连
        ↓ 失败
Tailscale 地址或 MagicDNS
        ↓
Tailscale 优先建立 P2P
        ↓ 失败
Tailscale DERP 加密中继
```

Tailscale 只负责组网，不负责 GoDesk 的远控身份认证。

### 4.2 Tailscale 使用边界

GoDesk：

- 不接入 Tailscale 管理 API；
- 不保存 Tailscale 登录令牌；
- 不要求用户提供 API Key；
- 只把 Tailscale 当作虚拟网卡和可达网络；
- 允许使用 `100.64.0.0/10` 地址或 MagicDNS 名称保存对端；
- 仍对所有 GoDesk 会话执行独立双向认证和加密。

Windows 防火墙只允许以下来源访问 GoDesk：

- 本地子网；
- Tailscale IPv4 网段 `100.64.0.0/10`；
- Tailscale IPv6 网段 `fd7a:115c:a1e0::/48`。

防火墙规则还必须绑定对应接口和 Windows 网络配置文件：本地子网规则只作用于专用网络，Tailscale 地址范围只作用于 Tailscale 虚拟接口，不能仅凭来源地址放行。GoDesk 不要求路由器端口映射，不应在普通公网接口上开放端口。

### 4.3 降级行为

- Tailscale 不可用时，同一局域网仍可连接；
- Tailscale 协调服务暂时不可用时，已经建立的直连会话尽量保持；
- 不能直连且 DERP 不可用时，明确报告网络不可达；
- 不自动退回到未加密公网连接。

## 5. 技术栈

- 语言与运行时：C#、.NET 10 LTS；
- 桌面界面：WinUI 3；
- GPU 图形：Direct3D 11；
- 屏幕捕获：DXGI Desktop Duplication；
- 视频处理：D3D11 Video Processor；
- 视频编码：Media Foundation H.264 MFT；
- 音频捕获和播放：WASAPI；
- 音频编码：Opus；
- 主传输协议：QUIC + TLS 1.3；
- 兼容传输协议：TCP + TLS 1.3；
- 控制消息编码：受限 CBOR；
- 本地 IPC：带 Windows ACL 的命名管道；
- 本地密钥：Windows CNG、TPM 和 DPAPI。
- 持久终端：Windows ConPTY；
- Codex 接入：A 端本地 STDIO MCP。

选择 C# 而非以 Rust 为主的原因：

- 第一版只支持 Windows；
- C# 可以直接使用 Windows Runtime、WinUI 和 .NET 服务模型；
- 减少 Rust、C++、C# 之间的 FFI 和 `unsafe` 边界；
- .NET 10 是处于支持期内的 LTS；
- H.264 等高性能路径主要由 GPU 和 Windows 系统组件执行。

## 6. 进程架构

```text
┌────────────────────────────────────────────┐
│ GoDesk.UI                                  │
│ 普通用户权限：设备列表、远控、文件与终端 UI│
└──────────────────┬─────────────────────────┘
                   │ 本地命名管道
┌──────────────────▼─────────────────────────┐
│ GoDesk.Agent                               │
│ 普通用户权限：截屏、编码、声音、输入与渲染 │
└──────────┬─────────────────────┬───────────┘
           │                     │
           │ 会话数据            │ 受限 IPC
┌──────────▼────────────┐  ┌─────▼──────────────┐
│ GoDesk.Transport      │  │ GoDesk.Broker      │
│ LocalService 权限     │  │ LocalSystem 权限   │
│ 网络监听、设备认证    │  │ 开机启动、登录会话 │
│ 加密传输、连接限速    │  │ Agent、Ctrl+Alt+Del│
└──────────┬────────────┘  └────────────────────┘
           │
      Tailscale/LAN
```

Codex MCP 扩展另外增加普通用户权限的 `GoDesk.McpBridge` 和 `GoDesk.TerminalHost`。它们不改变 Transport 与 Broker 的权限边界，详细职责见扩展规格。

### 6.1 GoDesk.UI

- 在普通用户权限下运行；
- 管理设备、配对和权限；
- 展示远端画面；
- 提供文件传输界面；
- 不要求日常以管理员身份启动。

### 6.2 GoDesk.Agent

- 每个活动 Windows 用户会话运行一个实例；
- 捕获、处理和编码本地桌面；
- 捕获系统声音；
- 解码和显示远端画面；
- 执行已授权的键鼠输入；
- 处理文件读写；
- 不持有 SYSTEM 权限。

### 6.3 GoDesk.Transport

- 使用 `LocalService` 账户；
- 开机运行；
- 监听 GoDesk 网络端口；
- 负责 TLS、设备认证、QUIC 和限速；
- 未认证连接不能创建媒体、文件或高权限 IPC；
- 不直接执行键鼠注入和文件写入。

### 6.4 GoDesk.Broker

- 是唯一使用 LocalSystem 的长期进程；
- 不开放网络端口；
- 仅接受专用 Service SID 可访问的命名管道；
- 只实现启动会话 Agent、发送安全注意序列、管理临时 SecureAgent 等少量命令；
- 对 IPC 消息类型、长度和调用者身份进行严格校验。

## 7. 设备身份与首次配对

### 7.1 设备身份

安装时生成 ECDSA P-256 设备密钥：

```text
设备 ID = Base32(SHA-256(设备公钥))
```

私钥策略：

- 优先通过 TPM 和 Windows CNG 保存为不可导出密钥；
- 无 TPM 时使用 Windows 软件密钥存储提供程序；
- 不提供导出私钥功能；
- 其他敏感配置使用 DPAPI 保护；
- 重装或删除私钥后必须重新配对。

### 7.2 配对流程

```text
1. 电脑 B 本地开启“允许配对”，有效期 5 分钟
2. 电脑 A 输入电脑 B 的 LAN、Tailscale 地址或 MagicDNS
3. 双方建立临时加密连接并交换公钥
4. 双方显示由公钥和握手记录派生的安全校验词
5. 用户在两台电脑上核对并确认
6. 双方保存对端证书、公钥指纹、名称和权限
7. 自动关闭配对入口
```

第一版不提供万能密码、远程找回和远程重置配对。设备关系丢失后需要在两台电脑本地重新配对。

## 8. 无人值守认证与授权

后续连接采用：

- TLS 1.3；
- 双向客户端证书；
- 对端证书公钥固定；
- 每次连接随机挑战；
- 禁止未配对证书进入应用协议；
- 禁止早期数据执行键鼠或文件操作。

不采用可被远程反复猜测的永久密码。

每个可信设备单独存储以下能力：

- 查看屏幕；
- 键盘和鼠标；
- 接收声音；
- 切换显示器；
- 向本机传文件；
- 从本机取文件；
- 浏览整个当前用户目录；
- 移动和重命名文件；
- 删除到回收站；
- 永久删除文件；
- 在远端电脑执行文件；
- 操作登录和安全桌面；
- 创建和控制普通持久终端；
- 请求创建管理员终端。

第一版采用“配对即完全信任”模型。首次配对成功后，除管理员终端外，上述权限默认全部开启，包括整个用户目录、永久删除、执行文件和普通终端。`admin-terminal` 是明确例外，只能在被控电脑本地开启，且每次创建仍需标准 UAC。

配对确认页必须完整列出默认权限和默认关闭的管理员终端，并显示以下含义：配对设备可以查看屏幕、控制键鼠、读取或删除当前用户文件，并以当前 Windows 用户身份执行程序和终端命令。管理员终端一旦在本地开启并通过 UAC，可能获得整机管理能力。用户必须在两台电脑上核对安全校验词并确认后，才能建立这种完全信任关系。

配对完成后，用户仍可在被控电脑本地关闭任意权限或撤销设备。权限只能在被控电脑本地修改，远端不能修改自己的授权范围，也不能阻止本地撤销。

即使无人值守已开启，本地仍然：

- 显示持续可见的连接状态；
- 显示当前控制设备；
- 提供全局快捷键立即断开；
- 保存本地审计记录；
- 不允许远端隐藏连接提示；
- 对失败认证和配对尝试执行指数退避。

## 9. 网络传输协议

### 9.1 端口与协商

- 默认 UDP 端口：`45830`；
- TCP 兼容端口：`45830`；
- QUIC ALPN：`godesk/1`；
- 主协议：QUIC + TLS 1.3；
- 回退协议：TCP + TLS 1.3。

### 9.2 逻辑通道

| 通道 | 特性 | 用途 |
|---|---|---|
| Control | 双向、可靠、高优先级 | 键鼠、显示器、心跳、权限状态 |
| Video | 单向、可终止和重建 | H.264 视频帧 |
| Audio | 单向、低延迟 | Opus 音频 |
| File | 每个任务独立双向流 | 文件元数据、数据块和断点续传 |
| Terminal | 每个终端独立、可靠、有序、可重连 | ConPTY 输入、输出、状态和窗口调整 |
| Telemetry | 双向、低频 | RTT、队列、丢帧和码率反馈 |

当视频积压超过约 150ms 时：

```text
终止旧 Video 流
      ↓
清空待编码和待解码旧帧
      ↓
生成新的 IDR 关键帧
      ↓
建立新的 Video 流
```

控制、音频和文件流不随视频流一起断开。

### 9.3 消息解析约束

- 所有消息包含协议版本、类型和长度；
- 控制消息使用确定长度、受限层级的 CBOR；
- 单条控制消息最大 64KiB；
- 字符串、集合和嵌套层数均设置上限；
- 未知或非法消息立即关闭对应连接；
- 每个连接、流和文件任务均有资源配额；
- 文件大小不决定内存分配大小；
- 协议错误不能进入 Broker 的高权限命令路径。

## 10. 屏幕捕获与视频链路

### 10.1 数据链路

```text
DXGI Desktop Duplication
          │ BGRA D3D11 Surface
          ▼
D3D11 Video Processor
          │ NV12
          ▼
Media Foundation H.264 Encoder
          │ H.264 Annex B
          ▼
QUIC Video Stream
          ▼
Media Foundation H.264 Decoder
          │ D3D11 Surface
          ▼
WinUI 3 SwapChainPanel
```

正常链路避免把完整画面复制到 CPU 内存。

### 10.2 视频格式

- H.264 Main Profile；
- 8-bit SDR；
- YUV 4:2:0；
- 不使用 B 帧；
- 最长 2 秒一个关键帧；
- 显示器切换、分辨率变化或流重建后立即生成 IDR；
- HDR 桌面通过 GPU 转换为 SDR；
- DRM 保护内容允许显示为黑色，不绕过操作系统保护。

优先选择 Media Foundation 硬件编码器。没有硬件编码器时使用软件编码，并限制在最高 `1080p 30fps`。

### 10.3 自适应档位

| 网络状态 | 分辨率与帧率 | 目标码率 |
|---|---:|---:|
| 局域网良好 | 1080p 60fps | 8–12Mbps |
| 普通互联网 | 1080p 30fps | 4–6Mbps |
| 网络较差 | 720p 30fps | 2–3Mbps |
| 严重拥塞 | 720p 15fps | 0.8–1.5Mbps |

控制端每秒反馈最后显示帧、画面排队延迟、解码耗时、丢帧和声音缓冲状态。

调整原则：

- 排队延迟超过 150ms 时快速降码率或帧率；
- 未恢复时降低分辨率；
- 连续稳定 10 秒后才逐级提升；
- 静态桌面根据脏区域降到约 2–5fps。

### 10.4 鼠标显示

鼠标位置和形状通过控制通道独立发送，在控制端本地绘制。输入映射使用显示器 ID、物理像素和归一化坐标，避免 DPI 或分辨率变化造成错位。

## 11. 多显示器

通过 `QueryDisplayConfig` 和 `DisplayConfigGetDeviceInfo` 获取显示器名称、设备路径、分辨率、刷新率、方向、主显示器状态和显卡适配器。

第一版只发送一个显示器：

```text
控制端选择显示器
        ↓
被控端验证显示器 ID
        ↓
停止当前捕获和编码
        ↓
切换 DXGI Output 和必要的显卡设备
        ↓
发送新尺寸、方向和 IDR 帧
```

目标切换时间为 0.5–1 秒。显示器被拔出时自动切回主显示器。

## 12. 远程声音

```text
WASAPI Loopback
      │ 48kHz stereo PCM
      ▼
Opus Encoder
      │ 20ms / 64–128kbps
      ▼
QUIC Audio Stream
      ▼
自适应抖动缓冲
      ▼
Opus Decoder → WASAPI Render
```

参数：

- 48kHz 双声道；
- 每包 20ms；
- 默认约 96kbps；
- 接收缓冲目标 60ms，在 40–120ms 间调整；
- 超过播放时间的数据直接丢弃；
- 网络恢复后重新同步音视频时间戳；
- 支持 Opus 丢包隐藏；
- 第一版只传系统声音。

## 13. 登录界面与安全桌面

普通桌面由 `GoDesk.Agent` 处理。

登录界面、锁屏或 UAC 安全桌面由 `GoDesk.Broker` 临时启动 `GoDesk.SecureAgent`：

- 仅允许经过双向认证且拥有相应权限的设备；
- 只提供低帧率画面、键鼠和安全注意序列；
- 不提供声音和文件；
- 不读取或记录用户输入的凭据；
- 离开安全桌面后立即退出；
- 与网络服务之间只通过受 ACL 保护的命名管道通信。

该能力是第一版最高风险的技术验证项：

1. 优先验证 DXGI Desktop Duplication；
2. 不可用时仅对登录界面尝试低帧率 GDI 捕获；
3. 不开发自定义 Credential Provider；
4. 如果目标系统安全策略阻止两种捕获方式，第一版将明确限制为用户已经登录后的无人值守控制。

## 14. 键盘与鼠标控制

### 14.1 控制端输入捕获

GoDesk 只在远控画布获得焦点且用户主动进入控制状态时捕获输入：

- 默认使用 Windows Raw Input 获取键盘和鼠标事件；
- 需要拦截本地快捷键时，只在远控画布处于控制状态期间安装焦点受限的键盘钩子；
- 画布失去焦点、会话暂停或连接断开后立即停止捕获；
- 不在后台长期监听键盘；
- 不记录按键内容；
- 不把输入事件写入日志或崩溃报告。

控制端使用 `Ctrl+Alt+Shift+G` 释放输入捕获。被控端使用 `Ctrl+Alt+Shift+F12` 立即断开远控会话。

Windows 保留且无法稳定捕获的组合键通过远控工具栏发送，例如：

- `Ctrl+Alt+Delete`；
- `Alt+Tab`；
- Windows 键；
- `Win+L`；
- `Ctrl+Shift+Esc`。

### 14.2 输入事件格式

每个输入事件包含：

- 会话 ID；
- 单调递增序号；
- 单调时钟时间戳；
- 事件类型；
- Windows 扫描码与扩展键标志；
- 按下、重复或释放状态；
- 鼠标按钮和滚轮增量；
- 目标显示器 ID；
- 归一化指针坐标。

键盘优先传输物理扫描码，使快捷键和按键位置与 Windows 远控习惯一致。第一版不提供独立的 Unicode 文本注入和剪贴板同步。

控制端根据远端画面实际显示区域处理黑边、缩放和旋转，将指针位置转换为 `0..65535` 的归一化坐标。被控端再次校验显示器 ID 和坐标范围后，映射为远端物理像素。

### 14.3 被控端输入注入

普通桌面由 `GoDesk.Agent` 通过 `SendInput` 注入输入。由于 Windows UIPI 限制，普通权限 Agent 不能向更高完整性级别的窗口注入输入；登录界面和 UAC 安全桌面必须走经过授权的 `GoDesk.SecureAgent`，不能用普通 `SendInput` 绕过。

被控端维护本次会话的按键按下集合，并在以下情况合成全部释放事件：

- 控制端窗口失去焦点；
- 控制模式暂停；
- 网络连接中断；
- 显示器切换；
- 会话结束。

这可以避免 Alt、Ctrl、Shift 或鼠标按钮在断线后保持按下。

### 14.4 输入安全限制

- 只有具有 `control-input` 权限的已配对设备可以发送输入；
- 每秒最多处理 2000 个输入事件；
- 序号重复、倒退或时间戳过旧的事件直接丢弃；
- 超过 1 秒仍未处理的鼠标移动和按键重复事件不再执行；
- `Ctrl+Alt+Delete` 只能通过 Broker 的窄接口请求；
- 第一版不允许远端屏蔽被控电脑的本地键鼠；
- 本地用户的输入与紧急断开快捷键始终具有优先权。

## 15. 文件传输

### 15.1 文件访问模型

GoDesk 使用本地配置的文件根目录。第一版支持将当前 Windows 用户目录作为共享根目录：

```text
接收目录：%USERPROFILE%\Downloads\GoDesk
用户目录：%USERPROFILE%
其他目录：默认无，由用户在被控电脑本地添加
```

每个共享目录只向远端暴露随机 `RootId` 和显示别名，不传输其真实绝对路径。远端只能：

- 浏览已共享目录；
- 从共享目录下载文件；
- 向接收目录或明确允许写入的共享目录上传；
- 新建传输所需的子目录；
- 在获得独立权限后访问整个当前用户目录；
- 在获得独立权限后移动、重命名、删除或执行文件。

“整个用户目录”能力在成功配对后默认开启，包括 Desktop、Documents、Downloads、Pictures 和 AppData 等当前用户可访问内容，但仍受 Windows ACL、文件占用和操作系统保护限制。用户可以在被控电脑本地关闭该能力。

远端不能浏览其他 Windows 用户的配置目录，也不能因为文件功能自动获得管理员或 SYSTEM 权限。

### 15.2 文件管理操作

第一版支持：

- 新建文件夹；
- 移动；
- 重命名；
- 复制；
- 删除到 Windows 回收站；
- 永久删除；
- 执行文件。

本节所列文件操作仍使用独立权限表达，以便用户以后单独关闭；它们在成功配对后默认全部开启。管理员终端的默认关闭规则不受本段影响。

删除默认使用 Windows `IFileOperation` 移入回收站，以便恢复。永久删除使用单独的 `permanent-delete` 权限，该权限在成功配对后默认开启，但控制端每次仍显示不可恢复确认。文件太大、目标卷不支持回收站或回收站不可用时，不自动降级为永久删除。

同卷移动优先使用重命名语义。跨卷移动采用“复制、校验、提交、删除源文件”的顺序；目标文件未完成校验前不得删除源文件。

移动、重命名和删除均只允许在已经授权的根目录内操作。源路径和目标路径分别执行完整的路径安全校验。

### 15.3 路径安全

所有远端路径都必须是相对于 `RootId` 的规范化相对路径。GoDesk 拒绝：

- `..` 路径穿越；
- 绝对路径；
- UNC 和设备路径；
- NTFS Alternate Data Streams；
- 符号链接、目录联接、挂载点和其他 reparse point；
- Windows 保留设备名；
- 空文件名、超长文件名和非法字符。

打开文件时使用 `FILE_FLAG_OPEN_REPARSE_POINT` 检查路径组成部分。获得文件句柄后，通过最终规范路径、卷标识和允许根目录再次验证，防止检查后替换和目录逃逸。

文件操作始终由普通用户权限的 Agent 执行，因此不能读取或写入当前 Windows 用户本身无权访问的位置。

OneDrive 等云文件占位符可能使用 reparse point。第一版不自动展开此类占位符；文件必须先在被控电脑上完成本地下载和实体化，才能远程传输或执行。

### 15.4 传输协议

每个传输任务使用随机 128-bit `TransferId` 和独立 QUIC 流。目录传输先发送受限清单，清单项包含：

- `RootId`；
- 规范化相对路径；
- 文件或目录类型；
- 文件大小；
- 修改时间；
- 传输属性；
- 内容校验信息。

文件使用固定 4MiB 数据块：

```text
ChunkIndex + Offset + Length + SHA-256 + Data
```

接收端将数据写入目标目录内的隐藏临时文件：

```text
.godesk-part-<TransferId>
```

每个数据块单独校验 SHA-256。全部接收后再校验完整文件 SHA-256，校验成功才关闭临时文件并在同一目录内完成最终重命名。临时文件与目标文件位于同一卷，避免跨卷移动破坏最终提交语义。

第一版只保留文件名、大小、目录结构和修改时间，不传输：

- NTFS ACL；
- 所有者；
- EFS 状态；
- ADS；
- 硬链接；
- 符号链接；
- 稀疏文件属性。

### 15.5 断点续传

接收端保存：

- 对端设备公钥指纹；
- TransferId；
- 清单摘要；
- 已验证数据块位图；
- 临时文件位置和长度；
- 最近更新时间。

重新连接后，只有原始配对设备、相同 TransferId 和相同清单摘要可以继续传输。发送端只补发未验证的数据块。

未完成任务保留 7 天，过期后由本地清理任务删除。用户可以在界面中立即取消并删除临时数据。

### 15.6 文件冲突

默认策略是保留两份：

```text
report.pdf
report (1).pdf
```

可选策略包括跳过、保留两份和覆盖。覆盖能力在成功配对后默认开启，但默认冲突策略仍为保留两份；只有控制端在具体任务中明确选择覆盖时，才替换已有文件。用户可以在被控电脑本地关闭覆盖权限，关闭后覆盖请求自动降级为保留两份。

文件最终提交使用同目录重命名。除非用户明确允许覆盖，否则不使用替换已有文件的标志。

### 15.7 远程执行文件

远程执行是独立的 `execute-file` 权限，并在成功配对后默认开启。配对确认页必须明确提示：该能力等价于允许控制端以当前 Windows 用户身份执行任意代码。用户可以在被控电脑本地关闭该权限。

执行规则：

- 只允许执行已经完成传输或原本存在于授权根目录内的文件；
- 不允许执行 `.godesk-part-*` 临时文件；
- 文件路径在启动前重新按句柄验证；
- 使用当前交互用户身份启动，而不是 LocalService 或 LocalSystem；
- 默认显示程序窗口，不提供隐藏执行；
- “执行文件”操作本身不接受自定义命令行参数和环境变量；
- 任意命令行、PowerShell、CMD 和 Conda 交互统一由独立的持久终端模块处理，遵循扩展规格中的 `terminal` 权限、ConPTY、重连和审计规则；
- GoDesk 不自动确认 UAC，也不自动请求 `runas`；
- 如果程序自行触发 UAC，只能通过已经单独授权的安全桌面控制处理；
- 返回启动成功、进程 ID 和退出状态，但不捕获进程标准输入输出。

控制端在每次执行前显示文件名、相对路径、大小和 SHA-256，并要求确认。被控端持续显示远程执行通知，并在本地审计日志中记录请求设备、文件路径、文件摘要、启动时间和结果。

需要明确：`execute-file` 与 `terminal` 都能产生当前用户级代码执行能力，但使用独立权限表达和独立协议路径。关闭其中一项不会隐式关闭另一项。

### 15.8 配额与调度

- 单个目录清单最多 100,000 项；
- 默认每个会话最多传输 100GiB；
- 目标磁盘至少保留 2GiB 可用空间；
- 同时最多执行 2 个文件任务；
- 每个任务最多保持 16MiB 未确认数据；
- 文件浏览每页最多返回 500 项；
- 文件流不得挤占控制、声音和视频所需带宽。

当画面排队延迟超过 150ms 时，文件调度器主动降低速率；持续拥塞时暂停发送新文件块。局域网模式允许用户手动解除带宽限制。

### 15.9 文件安全与用户提示

- 接收后的文件不会自动打开或执行；
- 可执行文件、脚本、快捷方式和安装包显示显著风险提示；
- 保留文件原始扩展名，不根据内容自动改名；
- 允许 Windows Defender 和其他本地安全软件正常扫描临时文件与最终文件；
- 文件传输日志默认只记录共享目录别名、相对路径、大小和结果，不记录文件内容；
- 文件传输权限与远控权限分离，可以只允许远控而禁止文件；
- 删除、永久删除、整个用户目录和执行文件分别使用独立权限；
- 成功配对后除 `admin-terminal` 外的权限默认开启，用户可以在被控电脑本地逐项关闭；
- 获得执行权限的可信设备被视为拥有当前 Windows 用户级控制能力。

## 16. 安全边界

本方案所称“安全优先”，首先保护的是设备身份、传输内容、权限边界和本机高权限进程不被未授权一方利用。根据本项目已确认的自用模型，配对成功等价于将另一台电脑设为完全可信设备；全部能力默认开启是明确的产品决策，不代表这些操作是低风险操作。

设计防御目标：

- 网络监听；
- 中间人；
- 恶意或失陷的中继；
- 密码猜测；
- 消息重放；
- 未配对设备；
- 畸形网络消息导致的越权；
- 低权限网络服务漏洞直接升级为 SYSTEM。

设计不防御：

- 任意一端已经存在管理员级恶意软件；
- 控制端被盗且系统处于解锁状态；
- 用户主动授权的控制端执行破坏性操作；
- 获得文件执行权限的可信设备以当前用户身份读取、修改或删除数据；
- 获得普通终端权限的可信设备以当前用户身份执行任意命令；
- 本地开启并通过 UAC 的管理员终端改变整机安全状态；
- 合法远控会话看到屏幕中已经展示的敏感信息。

## 17. 用户界面与交互流程

### 17.1 界面原则

GoDesk 第一版只服务两台自用电脑，界面以单一可信设备为中心，不设计企业设备分组、账号组织、工单和云端控制台。

交互原则：

- 默认显示中文，并跟随 Windows 系统语言；
- 日常运行不要求管理员权限；
- 高风险操作显示结果和影响，不使用含义模糊的“确定”；
- 网络状态、控制状态和文件执行状态始终可见；
- 错误信息同时说明发生了什么以及用户可以做什么；
- 支持高 DPI、键盘导航、屏幕阅读器和 Windows 高对比度模式；
- 不使用纯颜色表达连接或风险状态。

### 17.2 应用结构

主窗口使用左侧导航：

```text
┌──────────────────────────────────────────────┐
│ GoDesk                                  设置 │
├──────────┬───────────────────────────────────┤
│ 主页     │                                   │
│ 文件     │            当前页面               │
│ 终端     │                                   │
│ 活动记录 │                                   │
│ 设置     │                                   │
├──────────┴───────────────────────────────────┤
│ 本机：已就绪    网络：Tailscale    服务：正常 │
└──────────────────────────────────────────────┘
```

页面职责：

- 主页：本机状态、可信设备、连接入口；
- 文件：本地与远端文件管理；
- 终端：持久 PowerShell、CMD、Conda 会话及 Codex 状态；
- 活动记录：远控、文件、执行和终端审计；
- 设置：网络、设备、权限、画质、文件、终端和诊断。

远控画面使用独立窗口，避免主窗口布局限制显示面积。

### 17.3 首次启动向导

首次启动依次完成：

1. 检查 Windows 版本、GPU、Media Foundation、WASAPI 和磁盘空间；
2. 检测 Tailscale 服务和虚拟地址；
3. 安装并启动 GoDesk Windows 服务；
4. 生成设备密钥和设备 ID；
5. 设置本机显示名称；
6. 配置 Windows 防火墙规则；
7. 引导与另一台电脑配对；
8. 执行网络、视频、声音和文件自检。

安装 Windows 服务和防火墙规则时单独触发一次 UAC。完成安装后，日常启动不再要求提升权限。

如果未安装 Tailscale：

- 不静默安装第三方软件；
- 提供官方下载安装入口；
- 允许用户先以“仅局域网模式”完成 GoDesk 设置；
- Tailscale 可用后自动检测，不要求重新安装 GoDesk。

### 17.4 配对交互

被控端点击“允许新设备配对”后进入 5 分钟配对窗口：

```text
允许配对：04:59
本机地址：100.x.x.x
设备名称：书房电脑
[复制地址] [取消配对]
```

控制端输入 LAN 地址、Tailscale 地址或 MagicDNS 名称后，双方显示：

- 对端设备名称；
- 设备 ID；
- 六个安全校验词；
- 配对后默认开启的权限以及默认关闭的管理员终端；
- “等价于当前 Windows 用户级控制能力”的风险说明。

用户必须完成以下操作：

1. 在两台电脑上核对六个安全校验词；
2. 勾选“我确认正在配对自己的另一台电脑”；
3. 使用 Windows Hello 确认；
4. 如果 Windows Hello 不可用，则在两边分别点击带设备名称的确认按钮。

配对按钮不能使用笼统的“确定”，应显示：

```text
[完全信任“卧室电脑”并完成配对]
```

任意一边取消、超时或校验失败，双方都不保存信任关系和临时密钥。

### 17.5 主页

主页顶部显示本机卡片：

```text
书房电脑
状态：可连接
设备 ID：GD-7K2M-9QFP-4TX8
局域网：192.168.1.20
Tailscale：100.x.x.x
系统服务：运行中
```

可信设备卡片显示：

```text
卧室电脑                         在线
最近连接：今天 16:32
地址：卧室电脑.tailnet-name.ts.net

[远程控制] [文件管理] [终端] [更多]
```

“更多”包含查看设备指纹、权限、连接测试和撤销设备，不放置永久删除文件等会话内操作。

连接过程使用明确阶段：

```text
正在解析设备地址
正在建立加密连接
正在验证设备证书
正在启动远端会话
正在等待第一帧
```

失败时停留在失败阶段，提供“重试”和对应诊断信息，不只显示错误代码。

### 17.6 会话类型

主页提供三种入口：

- 远程控制：启动画面、声音、键鼠和文件能力；
- 文件管理：只建立控制和文件通道，不启动屏幕捕获、编码和声音；
- 终端：连接或创建持久终端，不启动屏幕捕获、编码和声音。

文件管理和终端模式可以减少资源占用。虽然除管理员终端外的配对权限默认开启，但未启动的会话类型不会创建不需要的媒体组件。

### 17.7 远控窗口

```text
┌────────────────────────────────────────────────────┐
│ 卧室电脑  Tailscale  32ms  1080p/30  4.8Mbps       │
│ [显示器1⌄] [画质⌄] [声音] [输入] [文件] [安全键] │
│                                      [全屏] [断开] │
├────────────────────────────────────────────────────┤
│                                                    │
│                    远端画面                        │
│                                                    │
└────────────────────────────────────────────────────┘
```

工具栏功能：

- 当前设备和连接类型；
- RTT、分辨率、帧率和估算码率；
- 显示器切换；
- 自动、清晰、流畅和自定义画质；
- 声音开关与音量；
- 键鼠控制开关；
- 打开文件管理窗口；
- `Ctrl+Alt+Delete`、`Alt+Tab`、Windows 键等安全键菜单；

全屏时工具栏自动收起，但顶部保留不可完全隐藏的连接状态条。鼠标移到顶部或按 `Ctrl+Alt+Shift+G` 后展开工具栏并释放输入捕获。

画面状态：

- 正常：显示实时画面；
- 切换显示器：显示短暂加载层，旧画面不拉伸；
- 网络抖动：保留最后一帧并显示“正在恢复连接”；
- 权限被本地关闭：立即停止相应功能并解释原因；
- 安全桌面：显示“安全桌面模式”，隐藏文件和声音入口。

### 17.8 被控电脑本地状态

被控电脑托盘图标状态：

- 灰色：服务运行、没有连接；
- 绿色：正在被远程控制；
- 蓝色：仅文件管理会话；
- 橙色：正在执行远端发起的程序；
- 红色：认证失败过多或服务异常。

连接建立后显示持续可见但尽量不遮挡内容的顶部状态条：

```text
“卧室电脑”正在控制此电脑    [暂停输入] [断开]
```

状态条可以缩小为顶部标记，但不能被远端彻底隐藏。被控端本地快捷键 `Ctrl+Alt+Shift+F12` 始终立即断开连接并释放所有远端输入状态。

远程执行程序时显示包含设备名称和文件名的本地通知。通知关闭不影响审计记录。

### 17.9 文件管理界面

文件页面采用双栏布局：

```text
┌──────────────────────┬──────────────────────┐
│ 本机                 │ 卧室电脑             │
│ 此电脑 > Downloads   │ 用户目录 > Documents │
├──────────────────────┼──────────────────────┤

├──────────────────────┴──────────────────────┤
│ 传输队列：进度、速度、剩余时间、暂停、取消 │
└─────────────────────────────────────────────┘
```

支持：

- 双击打开文件夹；
- 拖放上传和下载；
- 新建文件夹；
- 复制、移动和重命名；
- 删除到回收站；
- `Shift+Delete` 请求永久删除；
- 执行远端文件；
- 暂停、恢复和取消传输；
- 断线后的继续传输。

删除到回收站显示普通确认；永久删除显示红色不可恢复确认，并要求再次核对文件名。批量永久删除显示文件数量和总大小。

执行文件前显示：

- 远端设备；
- 文件名和相对路径；
- 文件大小；
- SHA-256；
- “将以远端当前 Windows 用户身份运行”的说明。

按钮文本使用：

```text
[在“卧室电脑”上运行此文件]
```

### 17.10 设置页面

设置分组：

1. 常规
   - 设备名称；
   - 开机启动；
   - 语言；
   - 关闭窗口后最小化到托盘。
2. 可信设备
   - 设备名称、ID、公钥指纹和配对时间；
   - 每项权限开关；
   - 撤销设备。
3. 网络
   - LAN 和 Tailscale 地址；
   - 监听端口；
   - 防火墙状态；
   - 连接自检。
4. 远控
   - 默认画质；
   - 声音；
   - 光标模式；
   - 本地快捷键。
5. 文件
   - 用户目录共享状态；
   - 接收目录；
   - 额外共享根目录；
   - 配额、临时文件保留期和带宽限制。
6. 终端
   - 普通终端权限和数量；
   - 管理员终端开关，默认关闭；
   - Codex MCP 配置状态；
   - 暂停 Codex 输入和结束全部终端。
7. 安全
   - 本机设备 ID 和指纹；
   - TPM 或软件密钥状态；
   - 配对开关；
   - 清除身份并重新初始化。
8. 诊断
   - 服务、捕获、编码、声音和 Tailscale 状态；
   - 导出脱敏诊断包。

“撤销设备”和“清除本机身份”属于破坏性操作。撤销后立即关闭相关会话；清除身份需要 Windows Hello 或本地管理员确认，并要求两台电脑重新配对。

### 17.11 活动记录

活动页面按时间倒序显示：

- 配对、撤销和权限变化；
- 连接开始、结束和异常断开；
- 控制会话或文件会话；
- 文件上传、下载、移动、覆盖和删除；
- 文件执行请求、SHA-256、进程 ID 和结果；
- 终端创建、连接、权限级别、操作摘要和退出状态；
- 认证失败和限速；
- 服务、Agent 和 SecureAgent 的关键状态变化。

日志仅保存在本机，默认保留 90 天或最多 50MiB，先到者触发轮换。导出诊断包默认隐藏用户绝对路径、Tailscale 地址和设备名称，用户可以在导出前预览包含内容。

### 17.12 断线与恢复体验

连接异常时：

1. 被控端立即释放所有远端按键和鼠标按钮；
2. 控制端保留最后一帧并显示重连层；
3. 按 1、2、4、8 秒退避，之后每 10 秒重试；
4. 最长自动重连 5 分钟，用户可以立即取消；
5. 每次重连重新执行设备证书认证；
6. 恢复后重新创建视频流并请求 IDR；
7. 文件任务按已验证数据块断点续传；
8. 远程执行请求绝不因重连自动重放。

自动重连失败后显示最终原因和诊断入口，不自动转为不安全的公网连接。

## 18. 安装、升级、卸载与异常恢复

### 18.1 安装包形式

第一版使用 x64、按机器安装的 MSI：

```text
GoDesk-Setup-x64.msi
```

安装技术采用 WiX Toolset。GoDesk.UI 使用 unpackaged WinUI 3，并将 .NET 10 与 Windows App SDK 依赖以 self-contained 方式随安装包分发。

选择该方式的原因：

- 可以安装和管理 Windows 服务；
- 可以创建受控的 Windows 防火墙规则；
- 不依赖目标电脑预装 .NET 或 Windows App SDK Runtime；
- 不要求启用 MSIX 侧载；
- 支持标准 MSI 修复、升级和卸载语义。

代价是安装包体积更大，而且 Windows App SDK 和 .NET 的安全更新必须通过发布新版 GoDesk 带入，不能依赖共享运行时自动更新。

### 18.2 安装内容

程序文件安装到：

```text
C:\Program Files\GoDesk\
```

机器级数据保存到：

```text
C:\ProgramData\GoDesk\
```

用户界面偏好保存到：

```text
%LOCALAPPDATA%\GoDesk\
```

主要数据布局：

```text
Program Files\GoDesk\
  GoDesk.UI.exe
  GoDesk.Agent.exe
  GoDesk.Transport.exe
  GoDesk.Broker.exe
  GoDesk.SecureAgent.exe
  GoDesk.McpBridge.exe
  GoDesk.TerminalHost.exe
  GoDesk.ShellAdapter.dll
  运行时与只读依赖

ProgramData\GoDesk\
  state.db
  Logs\
  Diagnostics\
  Updates\

LocalAppData\GoDesk\
  ui.json
  Cache\
```

设备私钥由 Windows CNG 保存，不作为普通文件写入 `ProgramData`。需要持久化的敏感配置使用 DPAPI 保护。

### 18.3 文件与目录 ACL

- `Program Files\GoDesk`：普通用户只读和执行，只有 Administrators、SYSTEM 和受信安装程序可写；
- `ProgramData\GoDesk`：仅 SYSTEM、GoDesk 服务 SID 和被明确授权的管理进程可写；
- `state.db`：UI 不直接打开，通过受认证的本地 IPC 查询；
- 诊断目录不允许远端网络会话直接写入；
- 接收和临时传输文件始终使用目标 Windows 用户权限；
- 安装目录不得包含任何普通用户可写的 DLL 搜索路径。

安装程序在完成复制后复核关键文件 ACL。如果 ACL 无法正确设置，安装失败并回滚，而不是以宽松权限继续运行。

### 18.4 Windows 服务生命周期

安装两个长期服务：

1. `GoDeskTransport`
   - 账户：`LocalService`；
   - 启动：Automatic；
   - 使用专用受限 Service SID；
   - 负责网络、TLS、QUIC 和本地会话路由。
2. `GoDeskBroker`
   - 账户：`LocalSystem`；
   - 启动：Automatic；
   - 不监听网络；
   - 只保留创建用户会话进程和安全桌面所需的最小权限。

Broker 先启动，Transport 可以在 Broker 暂时不可用时保持低权限网络服务运行，但不能执行安全桌面和会话创建操作。

服务恢复策略：

- 第一次异常退出后延迟 5 秒重启；
- 第二次延迟 30 秒；
- 10 分钟内第三次异常退出后停止自动重启；
- 在 Windows 事件日志和 GoDesk 活动记录中写入非敏感故障信息；
- 不形成无限高速崩溃循环。

### 18.5 Agent 启动与退出

Broker 监听 Windows 会话登录、解锁、锁定和注销事件：

- 用户登录后启动对应的 `GoDesk.Agent`；
- 同一会话只允许一个 Agent；
- Agent 必须运行在目标用户令牌和交互桌面中；
- 用户注销后终止该会话 Agent；
- 登录或安全桌面只按已认证会话需要启动临时 SecureAgent；
- SecureAgent 离开安全桌面后立即退出。

Agent 与 Transport 或 Broker 的命名管道断开时，立即停止捕获、文件操作和远端输入，并释放全部按键状态。

### 18.6 防火墙规则

安装程序创建 GoDesk 专用入站规则：

- UDP `45830`，作用域为 LocalSubnet 和 Tailscale 地址范围；
- TCP `45830`，仅用于兼容回退，作用域相同；
- 规则绑定 GoDesk.Transport 程序路径；
- 公用网络配置文件默认不允许普通公网来源；
- 卸载时按规则唯一 ID 删除，不按模糊名称批量删除。

如果 Tailscale 未安装，规则仍保留 LocalSubnet 范围。检测到 Tailscale 后由本地管理流程补充对应地址范围，不开放 `Any` 来源。

### 18.7 代码签名

所有可执行文件、DLL 和 MSI 都必须使用 Authenticode 签名，并附带可信时间戳。Transport 在启动 Agent、Broker 或 SecureAgent 前验证：

- 文件签名有效；
- 发布者证书与安装时固定的发布者一致；
- 文件位于受保护的安装目录；
- 文件版本与当前安装版本兼容。

个人自用阶段采用本地私有代码签名根证书，手动安装到两台电脑的受信任发布者存储中，以避免购买公网代码签名证书。签名私钥离线保存并使用强密码保护。

如果未来向其他用户分发，则改用公众信任的代码签名证书或 Microsoft Store 签名。不得发布未签名的正式安装包。

### 18.8 更新策略

第一版不实现静默自动更新，也不部署更新服务器。更新流程为：

1. 用户获取新的签名 MSI；
2. GoDesk 或安装程序通过 `WinVerifyTrust` 验证 Authenticode；
3. 核对发布者证书、公钥固定和版本号；
4. 阻止低版本包覆盖高版本，除非管理员显式进入恢复模式；
5. 停止接受新会话；
6. 提示并结束当前远控，暂停可续传文件任务；
7. 备份状态数据库并停止服务；
8. MSI 执行原位升级和配置迁移；
9. 重启服务并运行健康检查；
10. 失败时由 MSI 回滚文件，并恢复升级前数据库备份。

更新不重新生成设备身份，不删除配对关系，也不改变权限默认值。

协议兼容策略：

- 当前版本至少兼容前一个正式版本；
- 两台电脑可以先后升级；
- 协议版本不兼容时拒绝会话并明确提示需要升级哪一端；
- 不在不兼容时退回到未认证旧协议。

### 18.9 状态数据库与迁移

`state.db` 使用 SQLite，保存：

- 可信设备公钥和权限；
- 设备显示名称与非秘密配置；
- 活动审计索引；
- 文件断点续传状态；
- 配置架构版本。

数据库不保存设备私钥、Tailscale 令牌或文件内容。

迁移规则：

- 每次迁移在事务中执行；
- 迁移前生成带版本号的本地备份；
- 迁移失败不启动网络监听；
- 不允许部分迁移状态继续提供无人值守服务；
- 备份最多保留最近两个成功版本；
- 数据库损坏时进入恢复模式，不自动创建空信任库并对外开放。

### 18.10 日志与崩溃数据

日志分为：

- 审计日志：保留 90 天或 50MiB；
- 技术日志：默认保留 7 天或 20MiB；
- Windows 事件日志：仅记录服务启动、停止、崩溃和升级结果。

禁止记录：

- 设备私钥和会话密钥；
- 键盘输入内容；
- 文件内容；
- TLS 明文；
- 屏幕帧和声音数据；
- Windows 登录凭据。

默认不生成包含进程堆内存的完整崩溃转储。诊断模式只生成最小转储，并在创建前提示其中仍可能包含敏感元数据。

### 18.11 异常恢复

#### Transport 异常

- 网络套接字关闭，对端进入重连；
- Agent 发现 IPC 断开后立即释放输入并停止媒体和文件操作；
- 服务恢复后必须重新执行设备证书认证；
- 未完成的执行文件请求不自动重放。

#### Agent 异常

- Transport 立即结束对应媒体和文件通道；
- Broker 在受限次数内重启 Agent；
- Transport 保存的按键状态快照交给新 Agent执行全键释放；
- 视频以新 IDR 帧恢复；
- 文件从已校验块继续。

#### UI 异常

- 控制端 UI 退出或崩溃后，Transport 在 2 秒内关闭该控制会话；
- 被控端释放输入；
- 后台不得在用户看不到的情况下继续远控；
- 独立的文件管理任务默认暂停，而不是继续执行删除或运行程序。

#### Broker 异常

- 普通桌面会话可以短时间继续；
- 新的安全桌面操作和会话 Agent 创建失败并明确报告；
- SCM 按恢复策略重启 Broker；
- Transport 不尝试自行提升为 SYSTEM 作为替代。

#### 断电或系统崩溃

- SQLite 使用事务和 WAL 恢复；
- 临时文件保持 `.godesk-part-*` 状态；
- 启动后校验续传元数据与实际文件长度；
- 不完整的移动操作根据提交标记恢复源文件或目标文件；
- 永久删除和执行请求不在启动后重放。

### 18.12 功能故障降级

- 硬件编码器失败：尝试软件 H.264，并降低到最高 1080p30；
- 音频捕获失败：远控继续，显示“声音不可用”；
- 当前显示器拔出：切换主显示器并生成新 IDR；
- 文件校验失败：单块最多重试 3 次，仍失败则停止任务；
- 磁盘空间不足：暂停传输，不删除已有用户文件腾空间；
- Tailscale 不可用：只尝试已允许的局域网地址；
- 设备证书不匹配：立即终止，不询问是否临时忽略；
- Broker 不可用：禁用安全桌面功能，不扩大其他进程权限；
- 状态数据库不可验证：进入只允许本地修复的安全模式。

所有降级都遵循“功能可减少，认证和权限不能降级”的原则。

### 18.13 卸载与数据清理

卸载必须：

1. 停止接受新连接；
2. 结束远控和文件会话并释放全部输入；
3. 停止并删除 GoDeskTransport 与 GoDeskBroker；
4. 删除 GoDesk 创建的防火墙规则；
5. 删除会话启动项和快捷方式；
6. 删除程序文件；
7. 不删除用户已经接收完成的文件。

卸载前提供两种本地选择：

- 保留身份与设置：保留 CNG 设备密钥、配对、权限和断点状态，便于重装；
- 完全清除：删除 CNG 私钥、状态数据库、日志、缓存、诊断包和未完成临时传输文件。

完全清除是默认选项。执行前必须提示：另一台电脑仍会暂时显示旧设备，但旧设备身份将永远无法再次认证，重新安装后必须重新配对。

如果用户从 Windows 设置直接卸载且无法显示选择界面，则默认完整清除 GoDesk 自身的身份和配置，但不删除已完成的用户文件。无法安全确认归属的临时文件只记录位置并提示用户，不递归删除宽泛目录。

## 19. 性能、测试与验收标准

### 19.1 测试代码与工具位置

项目不创建或使用 `test/`、`tests/` 目录。`.gitignore` 必须持续包含：

```gitignore
test/
```

验证代码按用途放置：

```text
verification/
  unit/
  protocol/
  integration/
  security/
  system/

benchmarks/
  capture/
  codec/
  transport/
  file-transfer/
```

验证程序不得连接真实用户的 GoDesk 身份、真实共享目录或日常使用的 Tailscale 网络。破坏性验证只能运行在临时目录、一次性虚拟磁盘或可恢复的虚拟机快照内。

### 19.2 参考硬件

性能验收至少使用以下基线：

- Windows 11 22H2 或更高版本；
- 6 核 x64 CPU；
- 16GiB 内存；
- 支持 H.264 硬件编码和解码的 Intel、AMD 或 NVIDIA GPU；
- 千兆局域网；
- 一个 1920×1080 60Hz 显示器。

低于参考硬件时允许自动降低帧率或分辨率，但不得降低认证、加密和文件校验要求。

普通互联网 `1080p 30fps` 验收使用基准网络剖面：可用带宽至少 `8Mbps`、RTT 不高于 `100ms`、随机丢包不高于 `1%`，并分别覆盖 Tailscale 直连和 DERP 中继。超出该剖面时，验收目标改为会话不中断且能按第 10.5 节规则自动降低码率、帧率或分辨率；不承诺在不可控公网条件下维持 `1080p 30fps`。

### 19.3 核心性能指标

#### 画面

| 指标 | 目标 |
|---|---:|
| 局域网活动画面 | 1080p 60fps |
| 普通互联网活动画面 | 1080p 30fps 自适应 |
| 局域网端到端画面延迟 | p50 ≤ 70ms，p95 ≤ 100ms |
| 捕获到编码完成 | p95 ≤ 20ms |
| 解码到提交显示 | p95 ≤ 20ms |
| 稳态视频排队延迟 | p95 ≤ 80ms |
| 静态 1080p 桌面视频流量 | ≤ 500Kbps，不含声音 |
| 显示器切换到首个有效帧 | p95 ≤ 1 秒 |

局域网延迟通过高帧率摄像机或带同步时间戳的专用测量画面验证，不能用主观感受代替。

#### 输入

| 指标 | 目标 |
|---|---:|
| 被控端收到事件到调用输入注入 | p95 ≤ 10ms |
| 控制事件持续处理能力 | 2,000 events/s，无失控积压 |
| 断线后释放全部按键 | ≤ 200ms |
| 连续 100 次连接和断开 | 无粘滞按键或鼠标按钮 |

网络传播时间不计入被控端输入处理指标。

#### 声音

| 指标 | 目标 |
|---|---:|
| 局域网声音端到端延迟 | p95 ≤ 150ms |
| 音视频同步偏差 | 绝对值 ≤ 80ms |
| 连续播放 4 小时 | 无持续爆音、累积漂移或缓冲增长 |
| 设备切换恢复 | ≤ 3 秒或明确报告不可用 |

#### 文件

| 指标 | 目标 |
|---|---:|
| 无媒体时局域网吞吐 | ≥ 同路径基准吞吐的 70% |
| 有媒体时画面排队 | 保持 ≤ 150ms，否则文件自动降速 |
| 断点续传 | 只重传未验证的 4MiB 数据块 |
| 内容完整性 | 所有成功任务最终 SHA-256 一致 |
| 10GiB 文件传输 | 无进程内存随文件大小线性增长 |

#### 资源

| 指标 | 目标 |
|---|---:|
| 1080p60 硬件编码 CPU 占用 | 参考硬件平均 ≤ 20% |
| 单端工作集 | 稳态 ≤ 600MiB |
| 空闲后台服务工作集 | 合计 ≤ 150MiB |
| 8 小时连续远控 | 工作集增长 ≤ 100MiB，无句柄持续增长 |
| 登录后 Agent 就绪 | p95 ≤ 5 秒 |
| 网络可用后 Transport 可连接 | p95 ≤ 10 秒 |
| 网络恢复后的会话重连 | p95 ≤ 10 秒 |

### 19.4 单元与属性验证

重点覆盖：

- 协议帧长度、版本和类型校验；
- CBOR 层级、集合长度和非法编码；
- 输入序号、时间戳、重复事件和全键释放；
- DPI、黑边、旋转和多显示器坐标映射；
- 码率状态机的快速降级与缓慢升级；
- 设备权限默认值、关闭和撤销；
- 文件名、相对路径和 Windows 保留名称；
- 路径穿越、UNC、设备路径、ADS 和 reparse point；
- 文件分块、摘要、位图和断点恢复；
- 冲突命名、覆盖、移动和回滚；
- 状态数据库迁移与失败回滚；
- 日志脱敏和大小轮换；
- 安装、升级和卸载决策状态机。

协议和路径解析器使用属性生成与变异输入进行持续验证。任何解析失败必须返回受控错误，不得导致未处理异常、无限循环或无界内存分配。

### 19.5 协议与认证验证

必须验证：

- 正确配对设备可以完成 TLS 1.3 双向认证；
- 未知、自签但未固定、过期或格式异常的证书被拒绝；
- 证书公钥与已保存指纹不符时立即终止；
- 已撤销设备的现有连接立即关闭，后续连接被拒绝；
- 重放旧挑战、旧握手和旧输入序号无效；
- 配对窗口关闭后不能继续提交配对请求；
- 配对安全校验词在握手内容变化时变化；
- 非法 ALPN、协议版本和能力协商不能触发兼容性降级；
- 未认证连接不能创建媒体、文件或 Broker IPC；
- 连接风暴和失败认证受到资源配额及退避限制。

协议模糊测试覆盖控制帧、文件清单、视频帧头、音频帧头和 IPC 消息。Broker IPC 需要单独验证调用者 SID 和每种命令的参数边界。

### 19.6 文件安全矩阵

文件验证至少包含：

- 0B、1B、4MiB−1、4MiB、4MiB+1、10GiB 文件；
- Unicode、emoji、组合字符、大小写冲突和长路径；
- 空目录和最多 100,000 项的目录清单；
- 同名文件的保留两份、跳过和覆盖；
- 删除到回收站与永久删除；
- 同卷移动与跨卷移动；
- 传输中断、断电、进程崩溃和磁盘写满；
- 单块损坏、完整摘要不匹配和重复数据块；
- `..`、绝对路径、UNC、设备路径和 ADS；
- 符号链接、目录联接、挂载点和检查后替换；
- OneDrive 未实体化占位符；
- 被占用、只读、无权限和被安全软件隔离的文件；
- 可执行文件确认、哈希显示、启动失败和 UAC 行为；
- 重连后不得重复执行或重复永久删除。

删除、覆盖、跨卷移动和执行验证只在一次性虚拟磁盘或虚拟机中运行。

### 19.7 媒体与硬件矩阵

至少覆盖：

- Intel、AMD、NVIDIA 各一种硬件编码器；
- 无硬件编码器的软件回退；
- 单显示器、双显示器和显示器热插拔；
- 横屏、竖屏、不同 DPI 和不同分辨率组合；
- HDR 桌面转换为 SDR；
- 集成显卡与独立显卡分别连接显示器；
- 默认声音设备切换；
- 没有声音设备、静音设备和受保护音频；
- 静态桌面、视频播放、快速滚动和全屏 DirectX 内容；
- 50 次连续显示器切换；
- 编码器设备丢失和驱动重置。

受 DRM 保护内容显示黑色属于预期行为，但应用不得崩溃或泄露未保护帧缓存。

### 19.8 网络矩阵

验证环境包括：

- 同一千兆局域网；
- Wi-Fi；
- Tailscale 直接连接；
- 强制 Tailscale DERP 中继；
- IPv4 与 IPv6；
- RTT 20、50、100、200ms；
- 丢包 0.1%、1%、3%、5%；
- 乱序、短时断网和带宽突降；
- QUIC 不可用后的 TCP/TLS 回退；
- Tailscale 不可用但 LAN 可达；
- 网络从 LAN 切换到 Wi-Fi 或 Tailscale。

网络损伤验证使用隔离的虚拟网络和流量仿真路由器，不修改用户日常网络配置。每次场景记录 RTT、吞吐、帧率、码率、排队延迟、音频缓冲和重连时间。

### 19.9 服务、安装和恢复矩阵

必须验证：

- 全新安装、修复安装和原位升级；
- 安装失败后的 MSI 回滚；
- 安装目录和 ProgramData ACL；
- 防火墙规则来源范围和卸载删除；
- Transport 确实运行于 LocalService；
- Broker 不监听任何网络端口；
- 10 分钟内连续崩溃触发停止策略；
- UI、Agent、Transport 和 Broker 分别被强制终止；
- 断电后的 SQLite WAL、临时文件和移动恢复；
- 保留身份卸载与完整清除卸载；
- 完整清除不会删除已完成用户文件；
- 旧版本、篡改包、错误发布者和无签名 MSI 被拒绝；
- 升级保留设备身份、配对和权限；
- 前后两个正式版本能够互联；
- 不兼容协议明确拒绝且不退回旧认证方式。

### 19.10 安全验证

发布前完成：

- 所有 EXE、DLL 和 MSI 的签名检查；
- 依赖锁定、许可证清单和 SBOM；
- 已知漏洞扫描；
- .NET 分析器和编译警告视为错误；
- P/Invoke、unsafe、服务权限和命名管道 ACL 专项审查；
- 网络监听端口和防火墙规则核对；
- 安装目录用户不可写验证；
- 日志、诊断包和崩溃数据敏感信息检查；
- 私钥不可导出和删除身份后的不可恢复验证；
- 配对、撤销、执行文件和永久删除的人工滥用场景复核。

不得在没有书面风险说明的情况下忽略高危或严重依赖漏洞。授权设备能够以当前用户身份执行代码属于产品明确能力，不作为漏洞；未授权设备获得该能力属于发布阻断问题。

### 19.11 长时间与循环验证

- 8 小时持续 1080p60 局域网远控；
- 24 小时空闲在线；
- 4 小时持续声音；
- 100 次连接和断开；
- 100 次网络中断和恢复；
- 50 次显示器切换；
- 100 个混合文件任务；
- 在文件接收、校验、提交、移动和删除各阶段注入崩溃；
- 服务重启后确认没有自动重放执行和永久删除请求。

每轮记录进程工作集、私有字节、句柄、线程、GPU 显存、编码延迟和未完成任务数，用趋势而不是单个快照判断泄漏。

### 19.12 第一版功能验收

第一版必须完成：

- 两台 Windows 11 电脑通过安全校验词完成首次配对；
- 配对后除管理员终端外的权限默认开启，且可在被控端逐项关闭和撤销；
- LAN 和 Tailscale 环境均能建立远控；
- 1080p60 LAN 与基准网络剖面下的 1080p30 互联网自适应达到性能目标；
- 键鼠、特殊组合键和紧急断开可用；
- 多显示器可在 1 秒内切换；
- 系统声音稳定传输；
- 完整用户目录浏览、上传、下载、移动、删除和执行可用；
- A 端 Codex 可通过本地 STDIO MCP 控制 B 的持久普通终端并读取文件；
- 管理员终端默认关闭，开启后每次创建都经过 UAC；
- 断点续传和最终 SHA-256 校验可靠；
- 服务开机运行，无人值守会话在支持的桌面状态下可用；
- 登录界面和安全桌面按第 13 节的原型验证结论交付，不夸大系统不支持的能力；
- 手动签名 MSI 可以安全升级和卸载；
- 所有审计、撤销和异常恢复行为符合设计。

### 19.13 发布阻断条件

出现任一情况不得发布 1.0：

- 未配对设备能够进入应用协议或触发 Broker；
- 被撤销设备仍可建立新会话；
- 路径验证可以逃出授权根目录；
- 文件成功状态下摘要不一致；
- 执行或永久删除请求会在重连、重启或升级后自动重放；
- 终端输入在重连、服务重启或升级后自动重放；
- 普通终端获得高完整性或 SYSTEM，或管理员终端绕过 UAC；
- Transport 以 SYSTEM 或管理员权限运行；
- Broker 对公网或 Tailscale 监听端口；
- 正式二进制或 MSI 未签名或签名校验失败；
- 卸载或清理可能递归删除宽泛用户目录；
- 断线后远端输入状态无法可靠释放；
- 8 小时耐久验证出现崩溃、明显资源泄漏或不可恢复失控；
- 存在未处理的严重或高危安全问题；
- 核心功能只在开发机上成功，未通过第二台真实 Windows 电脑验证。
